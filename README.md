## О проекте

**AI Media Organizer** — это автоматизированная система для организации больших медиа-архивов с использованием компьютерного зрения и машинного обучения. Программа сканирует ваши фотографии и видео, распознаёт лица, группирует их по людям, удаляет дубликаты и автоматически раскладывает файлы по структурированным папкам.

Система работает **полностью локально** — все данные остаются на вашем компьютере, никакие файлы не отправляются в облако. Используются современные нейросетевые модели (YOLOv8, FaceNet) для точного распознавания лиц и кластеризации.

### Ключевые особенности

- **Распознавание лиц** — детекция лиц на фотографиях с помощью YOLOv8-Face и генерация уникальных векторных представлений (embeddings) через FaceNet
- **Автоматическая кластеризация** — группировка похожих лиц с использованием алгоритма HDBSCAN, интерактивное именование кластеров через графический интерфейс
- **Умная дедупликация** — удаление точных дубликатов (MD5) и визуально похожих изображений (perceptual hashing)
- **Геолокация** — извлечение GPS-координат из EXIF и обратное геокодирование (страна, город) в офлайн-режиме
- **Фильтрация скриншотов** — автоматическое определение снимков экрана по ключевым словам, EXIF-тегам и соотношению сторон
- **Гибкая организация** — копирование файлов в структурированные папки по людям, локациям, датам и группам
- **Поддержка форматов** — JPG, PNG, HEIC/HEIF, RAW (CR2, NEF, ARW, DNG), видео (MP4, MOV, AVI, MKV)

## Возможности

### 🎯 Автоматическое распознавание лиц

Система использует **YOLOv8-Face** для детекции лиц с точностью до пикселя и **FaceNet (InceptionResnetV1)** для генерации 512-мерных векторных представлений каждого лица. Это позволяет находить одного и того же человека на разных фотографиях, даже если освещение, угол съёмки или выражение лица отличаются.

**Инкрементальное обучение** — система запоминает именованные лица и автоматически распознаёт их на новых фотографиях при повторных запусках. Вы называете кластер один раз, и программа применяет это имя ко всем будущим снимкам этого человека.

**Фильтрация шума** — встроенная система негативных шаблонов ("Noise", "пустышка") автоматически отсеивает ложные срабатывания (статуи, постеры, отражения) с порогом сходства 0.80.

### 🗂️ Умная организация файлов

Программа автоматически раскладывает медиа-файлы по **пяти категориям**:

- **Пейзажі та Архітектура** — фотографии без лиц (0 faces) → `Пейзажі та Архітектура/Рік/Локація/`
- **Випадкові перехожі** — снимки с незнакомцами (только "пустышка" или "Noise") → `Випадкові перехожі/Рік/Локація/`
- **Групові фото** — снимки с 3+ лицами → `Групові фото/Рік/Локація/`
- **Люди** — фотографии с 1-2 распознанными лицами → `Люди/Ім'я_людини/Рік/` (копируется в папку каждого человека)
- **Невідомі особи** — снимки с 1-2 нераспознанными лицами (кластеры `Cluster_X`) → `Невідомі особи/Рік/Локація/`

**Кошик (Trash)** — отдельная папка для скриншотов, визуальных дубликатов и MD5-дубликатов с подпапками `Скріншоти та Меми`, `Візуальні дублікати`, `Дублікати MD5`.

### 🔍 Дедупликация на трёх уровнях

1. **MD5-хеширование** — точное совпадение байтов файла (100% дубликаты)
2. **Perceptual hashing** — визуальное сходство изображений с порогом расстояния Хэмминга (по умолчанию 4)
3. **Внутрикластерная дедупликация** — удаление дублирующихся лиц внутри одного кластера (cosine similarity ≥ 0.95)

### 📍 Офлайн-геокодирование

Извлечение GPS-координат из EXIF-метаданных и преобразование их в названия стран и городов **без интернета** с помощью библиотеки `reverse_geocoder`. Поддержка украинских названий стран через файл `geo_overrides.json`.

### 🖼️ Графический интерфейс для именования

**Веб-интерфейс на базе Eel** для удобного просмотра и именования кластеров лиц:

- Превью всех лиц в кластере в виде удобной и плавной адаптивной сетки
- Поле ввода имени с автодополнением из ранее введённых имён
- Кнопки быстрых действий ("Noise") для быстрой фильтрации ложных детекций
- Интерактивный прогресс-бар и логи выполнения всех этапов конвейера прямо в браузере

## Установка

### Шаг 1: Клонирование репозитория

```bash
git clone https://github.com/yourusername/ai-media-organizer.git
cd ai-media-organizer/organizer
```

### Шаг 2: Создание виртуального окружения

**Linux/macOS:**
```bash
python3 -m venv .venv
source .venv/bin/activate
```

**Windows (PowerShell):**
```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

**Windows (CMD):**
```cmd
python -m venv .venv
.venv\Scripts\activate.bat
```

### Шаг 3: Установка зависимостей

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

**Зависимости:**
- `torch` + `torchvision` — PyTorch для нейросетей (автоматически устанавливается с `facenet-pytorch`)
- `ultralytics` — YOLOv8-Face для детекции лиц
- `facenet-pytorch` — FaceNet (InceptionResnetV1) для генерации embeddings
- `hdbscan` — алгоритм кластеризации HDBSCAN
- `opencv-python` — обработка изображений
- `pillow-heif` — поддержка HEIC/HEIF
- `rawpy` — поддержка RAW-форматов (CR2, NEF, ARW, DNG)
- `exifread` — извлечение EXIF-метаданных
- `reverse_geocoder` — офлайн-геокодирование
- `imagehash` — perceptual hashing для дедупликации
- `eel` — веб-интерфейс
- `tqdm` — прогресс-бары

### Шаг 4: Автоматическая загрузка моделей

При первом запуске программа автоматически скачает:

- **YOLOv8-Face** (~6 МБ) — веса модели детекции лиц с HuggingFace (junjiang/GestureFace)
- **FaceNet (vggface2)** (~100 МБ) — предобученные веса InceptionResnetV1 через `facenet-pytorch`

Модели сохраняются в `organizer/data/models/` и больше не скачиваются при последующих запусках.

## Системные требования

### Минимальные требования

- **ОС:** Windows 10/11, Linux (Ubuntu 20.04+), macOS 11+
- **Python:** 3.8 или выше
- **RAM:** 4 ГБ (для CPU-режима)
- **Диск:** 2 ГБ свободного места (модели + база данных)
- **Процессор:** Intel Core i5 / AMD Ryzen 5 или аналог

### Рекомендуемые требования

- **ОС:** Windows 11 / Ubuntu 22.04 / macOS 12+
- **Python:** 3.10+
- **RAM:** 8 ГБ (для CUDA-режима)
- **GPU:** NVIDIA с поддержкой CUDA 11.8+ и 4+ ГБ VRAM (GTX 1650, RTX 3050 или выше)
- **Диск:** 10 ГБ свободного места (для больших архивов)
- **Процессор:** Intel Core i7 / AMD Ryzen 7 или выше

### Производительность

**CPU-режим (без GPU):**
- ~2-5 секунд на фото (детекция + embedding)
- Архив из 1000 фото обрабатывается за ~40-80 минут

**CUDA-режим (с GPU):**
- ~0.2-0.5 секунды на фото
- Архив из 1000 фото обрабатывается за ~5-10 минут

**Примечание:** Время зависит от количества лиц на фотографиях. Групповые снимки обрабатываются дольше.

## Использование

### Быстрый старт

1. Поместите ваши фотографии и видео в папку `input/` (создастся автоматически)
2. Запустите программу:

```bash
python main.py
```

3. Дождитесь завершения 5 этапов конвейера (сканирование → метаданные → лица → кластеризация → организация)
4. В графическом окне назовите кластеры лиц или отметьте их как "Noise"/"пустышка"
5. Организованные файлы появятся в папке `output/`

**Указать свои папки:**
```bash
python main.py --source /path/to/photos --dest /path/to/organized
```

```bash
python main.py --device cpu
```

**Сбросить базу данных и начать заново:**
```bash
python main.py --reset-db
```

**Изменить порог уверенности детекции лиц:**
```bash
python main.py --face-conf 0.7
```

**Сохранить текущие параметры в config.json:**
```bash
python main.py --source ./photos --dest ./archive --save-settings
```

## Как это работает

### Архитектура конвейера

Программа выполняет обработку в **5 последовательных этапов**:

#### Этап 1: Сканирование и дедупликация

**Модуль:** `media_parser.py` → `scan_and_deduplicate()`

1. Рекурсивное сканирование папки `input/` с поиском медиа-файлов (JPG, PNG, HEIC, RAW, MP4, MOV и др.)
2. Вычисление **MD5-хеша** для каждого файла — точные дубликаты помечаются как `status="duplicate"`
3. Генерация **perceptual hash** (pHash) для изображений — визуально похожие файлы с расстоянием Хэмминга ≤ 4 помечаются как дубликаты
4. Сохранение метаданных в таблицу `media` базы данных SQLite

**Результат:** База данных с уникальными файлами, дубликаты помечены для удаления.

#### Этап 2: Извлечение метаданных и фильтрация

**Модуль:** `media_parser.py` → `extract_metadata_and_filter()`

1. Чтение **EXIF-тегов** из каждого файла (дата съёмки, GPS-координаты, модель камеры, софт)
2. **Обратное геокодирование** GPS → (страна, город) через `reverse_geocoder` в офлайн-режиме
3. **Детекция скриншотов** по трём критериям:
   - Ключевые слова в имени файла: "screenshot", "screen", "скриншот", "снимка"
   - EXIF-тег `Software` содержит названия программ для скриншотов
   - Соотношение сторон 1.7-2.2 (типичные пропорции экранов смартфонов)
4. Обновление полей `creation_date`, `latitude`, `longitude`, `country`, `city`, `status` в БД

**Результат:** Полная метаинформация для каждого файла, скриншоты помечены как `status="screenshot"`.

#### Этап 3: Детекция лиц и генерация embeddings

**Модуль:** `face_processor.py` → `process_faces()`

1. **YOLOv8-Face** сканирует каждое изображение и находит bounding boxes всех лиц с уверенностью ≥ 0.6
2. Каждое лицо вырезается из изображения и нормализуется до 160×160 пикселей
3. **FaceNet (InceptionResnetV1)** генерирует **512-мерный вектор** (embedding) для каждого лица
4. Embeddings сохраняются в таблицу `faces` как BLOB (numpy float32), привязываются к `media_id`

**Результат:** Таблица `faces` с векторными представлениями всех обнаруженных лиц.

#### Этап 4: Кластеризация и именование

**Модуль:** `clustering.py` → `cluster_and_name_faces()`

**Шаг 4.1: Негативная фильтрация**
- Новые лица сравниваются с негативными шаблонами ("Noise", "пустышка") через cosine similarity
- Совпадения с порогом ≥ 0.80 автоматически отсеиваются

**Шаг 4.2: Инкрементальное распознавание**
- Оставшиеся лица сравниваются с именованными кластерами (порог 0.75)
- Совпадения автоматически получают имя существующего кластера

**Шаг 4.3: HDBSCAN-кластеризация**
- Только новые лица (без `cluster_id`) кластеризуются алгоритмом HDBSCAN
- Параметры: `min_cluster_size=3`, `metric='euclidean'`
- Каждому кластеру присваивается уникальный ID `Cluster_N`

**Шаг 4.4: Внутрикластерная дедупликация**
- Внутри каждого кластера удаляются дублирующиеся лица (similarity ≥ 0.95)
- Предотвращает появление одного и того же лица 3-5 раз в одном кластере

**Шаг 4.5: Графический веб-интерфейс**
- Eel-интерфейс в браузере показывает превью всех лиц в новых кластерах
- Пользователь вводит имя или нажимает "Noise"/"пустышка"
- Имена сохраняются в поле `cluster_id` таблицы `faces`

**Результат:** Все лица распределены по именованным кластерам или отфильтрованы как шум.

#### Этап 5: Организация файлов

**Модуль:** `file_organizer.py` → `organize_files()`

1. Чтение всех записей из таблицы `media` с их статусами и связанными лицами
2. **Логика распределения:**
   - 0 лиц → `Пейзажі та Архітектура/Рік/Локація/`
   - Только "пустышка"/"Noise" → `Випадкові перехожі/Рік/Локація/`
   - ≥3 лица → `Групові фото/Рік/Локація/`
   - 1-2 распознанных лица → `Люди/Ім'я/Рік/` (копируется в папку каждого человека)
   - 1-2 нераспознанных лица (`Cluster_X`) → `Невідомі особи/Рік/Локація/`
   - Скриншоты → `Кошик/Скріншоти та Меми/`
   - Дубликаты → `Кошик/Візуальні дублікати/` или `Кошик/Дублікати MD5/`
3. **Физическое копирование** файлов с проверкой MD5-хеша после копирования
4. Создание иерархии папок `Рік/Локація/` или `Рік/` в зависимости от наличия GPS-данных

**Результат:** Структурированный архив в папке `output/` с файлами, разложенными по категориям.

### База данных SQLite

**Таблица `media`:**
- `id` — уникальный идентификатор
- `filepath` — полный путь к исходному файлу
- `filename` — имя файла
- `filesize` — размер в байтах
- `media_type` — "image" или "video"
- `creation_date` — Unix timestamp из EXIF
- `md5_hash` — MD5-хеш содержимого
- `image_hash` — perceptual hash (pHash)
- `status` — "keep", "duplicate", "screenshot"
- `latitude`, `longitude` — GPS-координаты
- `country`, `city` — геокодированные названия

**Таблица `faces`:**
- `id` — уникальный идентификатор
- `media_id` — внешний ключ на `media.id`
- `bbox` — JSON с координатами bounding box `{"x1": ..., "y1": ..., "x2": ..., "y2": ...}`
- `face_embedding` — BLOB с 512 float32 (2048 байт)
- `cluster_id` — имя кластера ("Іван", "Марія") или "Cluster_N", "Noise", "пустышка"

### Алгоритмы машинного обучения

**YOLOv8-Face:**
- Архитектура: YOLOv8n (nano) с модификацией для детекции лиц
- Входные данные: RGB-изображение произвольного размера
- Выходные данные: bounding boxes + confidence scores
- Скорость: ~100-200 FPS на GPU, ~5-10 FPS на CPU

**FaceNet (InceptionResnetV1):**
- Архитектура: Inception-ResNet v1 с triplet loss
- Входные данные: 160×160 RGB-изображение лица
- Выходные данные: 512-мерный L2-нормализованный вектор
- Предобучение: VGGFace2 (3.3M изображений, 9131 человек)

**HDBSCAN:**
- Иерархическая кластеризация на основе плотности
- Автоматическое определение количества кластеров
- Устойчивость к шуму (outliers помечаются как `-1`)
- Метрика: Euclidean distance для 512-мерных векторов

## Конфигурация

### Файл config.json

```json
{
  "source_dir": "input",
  "dest_dir": "output",
  "db_path": "data/database.db",
  "yolo_weights": "data/models/yolov8n-face.pt",
  "facenet_weights": "vggface2",
  "device": "cuda",
  "face_confidence": 0.6,
  "metadata_threads": 8,
  "screenshot_rules": {
    "keywords": ["screenshot", "screen", "скриншот", "снимка"],
    "check_exif": true,
    "aspect_ratio_min": 1.7,
    "aspect_ratio_max": 2.2
  },
  "duplicates": {
    "hash_size": 8,
    "hash_threshold": 4
  }
}
```

| `--save-settings` | Сохранить параметры в config.json | `--save-settings` |
| `--reset-db` | Удалить БД и начать заново | `--reset-db` |

**Приоритет настроек:** CLI-аргументы > config.json > дефолтные значения

### Настройка дедупликации

**Параметр `hash_threshold`** (по умолчанию 4):
- **0-2** — очень строгая дедупликация (только почти идентичные изображения)
- **3-5** — умеренная (рекомендуется, отсеивает кадры из серийной съёмки)
- **6-10** — агрессивная (может удалить похожие, но разные снимки)

### Настройка детекции скриншотов

**Параметр `aspect_ratio_min` / `aspect_ratio_max`:**
- Для вертикальных экранов смартфонов: 1.7-2.2 (по умолчанию)
- Для горизонтальных экранов ноутбуков: 1.3-1.8
- Отключить проверку соотношения сторон: установите оба значения в 0

## Часто задаваемые вопросы

**Q: Программа не видит мою GPU, хотя CUDA установлена.**  
A: Убедитесь, что версия PyTorch совместима с вашей версией CUDA. Проверьте: `python -c "import torch; print(torch.cuda.is_available())"`. Если `False`, переустановите PyTorch с правильной версией CUDA с сайта pytorch.org.

**Q: Детекция лиц пропускает некоторые лица.**  
A: Понизьте порог уверенности: `--face-conf 0.4`. Учтите, что это увеличит количество ложных срабатываний (статуи, постеры).

**Q: Программа создаёт слишком много кластеров для одного человека.**  
A: Это может происходить из-за разного освещения или возраста на фотографиях. После первого запуска назовите все кластеры одного человека одинаковым именем — при следующем запуске система объединит их автоматически.

**Q: Можно ли запустить программу без графического интерфейса?**  
A: Да, для полностью автоматического режима вы можете запустить конвейер напрямую, импортировав функции из модулей `src/`. При этом кластеры останутся неименованными (`Cluster_N`).

**Q: Сколько места занимает база данных?**  
A: Примерно 2-3 КБ на каждое фото (метаданные + embeddings всех лиц). Для архива из 10,000 фото — около 20-30 МБ.

**Q: Программа зависла на этапе кластеризации.**  
A: HDBSCAN может работать долго на больших датасетах (>10,000 лиц). Проверьте логи в `data/logs/organizer.log`. Если процесс действительно завис, перезапустите с `--reset-db`.
