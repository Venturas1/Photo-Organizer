## Про проєкт

**AI Media Organizer** — це автоматизована система для організації великих медіаархівів з використанням комп'ютерного зору та машинного навчання. Програма сканує ваші фотографії та відео, розпізнає обличчя, групує їх по людях, видаляє дублікати та автоматично розкладає файли по структурованих папках.

Система працює **повністю локально** — всі дані залишаються на вашому комп'ютері, жодні файли не надсилаються в хмару. Використовуються сучасні нейромережеві моделі (YOLOv8, FaceNet) для точного розпізнавання облич та кластеризації.

### Ключові особливості

- **Розпізнавання облич** — детекція облич на фотографіях за допомогою YOLOv8-Face та генерація унікальних векторних представлень (embeddings) через FaceNet
- **Автоматична кластеризація** — групування схожих облич з використанням алгоритму HDBSCAN, інтерактивне іменування кластерів через графічний інтерфейс
- **Розумна дедуплікація** — видалення точних дублікатів (MD5) та візуально схожих зображень (perceptual hashing)
- **Геокартування** — вилучення GPS-координат з EXIF та зворотне геокодування (країна, місто) в офлайн-режимі
- **Фільтрація скріншотів** — автоматичне визначення знімків екрана за ключовими словами, EXIF-тегами та співвідношенням сторін
- **Гнучка організація** — копіювання файлів у структуровані папки за людьми, локаціями, датами та групами
- **Підтримка форматів** — JPG, PNG, HEIC/HEIF, RAW (CR2, NEF, ARW, DNG), відео (MP4, MOV, AVI, MKV)

## Можливості

### 🎯 Автоматичне розпізнавание облич

Система використовує **YOLOv8-Face** для детекції облич із точністю до пікселя та **FaceNet (InceptionResnetV1)** для генерації 512-вимірних векторних представлень кожного обличчя. Це дозволяє знаходити одну й ту саму людину на різних фотографіях, навіть якщо освітлення, кут зйомки чи вираз обличчя відрізняються.

**Інкрементне навчання** — система запам'ятовує іменовані обличчя та автоматично розпізнає їх на нових фотографіях при повторних запусках. Ви називаєте кластер один раз, і програма застосовує це ім'я до всіх майбутніх знімків цієї людини.

**Фільтрація шуму** — вбудована система негативних шаблонів ("Noise", "пустышка") автоматично відсіює помилкові спрацьовування (статуї, постери, відображення) з порогом схожості 0.80.

### 🗂️ Розумна організація файлів

Програма автоматично розкладає медіафайли за **п'ятьма категоріями**:

- **Пейзажі та Архітектура** — фотографії без облич (0 faces) → `Пейзажі та Архітектура/Рік/Локація/`
- **Випадкові перехожі** — знімки з незнайомцями (тільки "пустышка" або "Noise") → `Випадкові перехожі/Рік/Локація/`
- **Групові фото** — знімки з 3+ обличчями → `Групові фото/Рік/Локація/`
- **Люди** — фотографії з 1-2 розпізнаними обличчями → `Люди/Ім'я_людини/Рік/` (копіюється в папку кожної людини)
- **Невідомі особи** — знімки з 1-2 нерозпізнаними обличчями (кластери `Cluster_X`) → `Невідомі особи/Рік/Локація/`

**Кошик (Trash)** — окрема папка для скріншотів, візуальних дублікатів та MD5-дублікатів з підпапками `Скріншоти та Меми`, `Візуальні дублікати`, `Дублікати MD5`.

### 🔍 Дедуплікація на трьох рівнях

1. **MD5-хешування** — точний збіг байтів файлу (100% дублікати)
2. **Perceptual hashing** — візуальна схожість зображень з порогом відстані Хеммінга (за замовчуванням 4)
3. **Внутрішньокластерна дедуплікація** — видалення дубльованих облич всередині одного кластера (cosine similarity ≥ 0.95)

### 📍 Офлайн-геокодування

Вилучення GPS-координат з EXIF-метаданих та перетворення їх на назви країн і міст **без інтернету** за допомогою бібліотеки `reverse_geocoder`. Підтримка українських назв країн через файл `geo_overrides.json`.

### 🖼️ Графічний інтерфейс для іменування

**Веб-интерфейс на базі Eel** для зручного перегляду та іменування кластерів облич:

- Прев'ю всіх облич у кластері у вигляді зручної та плавної адаптивної сітки
- Поле введення імені з автодоповненням із раніше введених імен
- Кнопки швидких дій ("Noise") для швидкої фільтрації помилкових детекцій
- Інтерактивний прогрес-бар та логи виконання всіх етапів конвеєра прямо в браузері

```cmd
python -m venv .venv
.venv\Scripts\activate.bat
```

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

**Залежності:**
- `torch` + `torchvision` — PyTorch для нейромереж (автоматично встановлюється з `facenet-pytorch`)
- `ultralytics` — YOLOv8-Face для детекції облич
- `facenet-pytorch` — FaceNet (InceptionResnetV1) для генерації embeddings
- `hdbscan` — алгоритм кластеризації HDBSCAN
- `opencv-python` — обробка зображень
- `pillow-heif` — підтримка HEIC/HEIF
- `rawpy` — підтримка RAW-форматів (CR2, NEF, ARW, DNG)
- `exifread` — вилучення EXIF-метаданих
- `reverse_geocoder` — офлайн-геокодування
- `imagehash` — perceptual hashing для дедуплікації
- `eel` — веб-інтерфейс
- `tqdm` — прогрес-бари

### Крок 4: Автоматичне завантаження моделей

При першому запуску програма автоматично завантажить:

- **YOLOv8-Face** (~6 МБ) — ваги моделі детекції облич з HuggingFace (junjiang/GestureFace)
- **FaceNet (vggface2)** (~100 МБ) — попередньо навчені ваги InceptionResnetV1 через `facenet-pytorch`

Моделі зберігаються в `organizer/data/models/` і більше не завантажуються при наступних запусках.

- **ОС:** Windows 11 / Ubuntu 22.04 / macOS 12+
- **Python:** 3.10+
- **RAM:** 8 ГБ (для CUDA-режиму)

- ~2-5 секунд на фото (детекція + embedding)

**CUDA-режим (з GPU):**
- ~0.2-0.5 секунди на фото
- Архів із 1000 фото обробляється за ~5-10 хвилин

**Примітка:** Час залежить від кількості облич на фотографіях. Групові знімки обробляються довше.

## Використання

### Швидкий старт

1. Помістіть ваші фотографії та відео в папку `input/` (створиться автоматично)
2. Запустіть програму:

```bash
python main.py
```

3. Дочекайтеся завершення 5 етапів конвеєра (сканування → метадані → обличчя → кластеризація → організація)
4. У графічному вікні назвіть кластери облич або позначте їх як "Noise"/"пустышка"
5. Організовані файли з'являться в папці `output/`

**Вказати свої папки:**
```bash
python main.py --source /path/to/photos --dest /path/to/organized
```

**Використовувати CPU замість GPU:**
```bash
python main.py --device cpu
```

**Скинути базу даних та почати заново:**
```bash
python main.py --reset-db
```

**Змінити поріг впевненості детекції облич:**
```bash
python main.py --face-conf 0.7
```

**Зберегти поточні параметри в config.json:**

```bash
python main.py --source ./photos --dest ./archive --save-settings
```

## Як це працює

### Архітектура конвеєра

Програма виконує обробку в **5 послідовних етапів**:

#### Етап 1: Сканування та дедуплікація

**Модуль:** `media_parser.py` → `scan_and_deduplicate()`

1. Рекурсивне сканування папки `input/` з пошуком медіа-файлів (JPG, PNG, HEIC, RAW, MP4, MOV та ін.)
2. Обчислення **MD5-хешу** для кожного файлу — точні дублікати позначаються як `status="duplicate"`
3. Генерація **perceptual hash** (pHash) для зображень — візуально схожі файли з відстанню Хеммінга ≤ 4 позначаються як дублікати
4. Збереження метаданих у таблицю `media` бази даних SQLite

**Результат:** База даних з унікальними файлами, дублікати позначені для видалення.

#### Етап 2: Вилучення метаданих та фільтрація

**Модуль:** `media_parser.py` → `extract_metadata_and_filter()`

1. Читання **EXIF-тегів** з кожного файлу (дата зйомки, GPS-координати, модель камери, софт)
2. **Зворотне геокодування** GPS → (країна, місто) через `reverse_geocoder` в офлайн-режимі
3. **Детекція скріншотів** за трьома критеріями:
   - Ключові слова в імені файлу: "screenshot", "screen", "скриншот", "снимка"
   - EXIF-тег `Software` містить назви програм для скріншотів
   - Співвідношення сторін 1.7-2.2 (типові пропорції екранів смартфонів)
4. Оновлення полей `creation_date`, `latitude`, `longitude`, `country`, `city`, `status` в БД

**Результат:** Повна метаінформація для кожного файлу, скріншоти позначені як `status="screenshot"`.

#### Етап 3: Детекція облич та генерація embeddings

**Модуль:** `face_processor.py` → `process_faces()`

1. **YOLOv8-Face** сканує кожне зображення та знаходить bounding boxes усіх облич з впевненістю ≥ 0.6
2. Кожне обличчя вирізається із зображення та нормалізується до 160×160 пікселів
3. **FaceNet (InceptionResnetV1)** генерує **512-вимірний вектор** (embedding) для кожного обличчя
4. Embeddings зберігаються в таблицю `faces` як BLOB (numpy float32), прив'язуються к `media_id`

**Результат:** Таблиця `faces` з векторними представленнями усіх виявлених облич.

#### Етап 4: Кластеризація та іменування

**Модуль:** `clustering.py` → `cluster_and_name_faces()`

**Крок 4.1: Негативна фільтрація**
- Нові обличчя порівнюються з негативними шаблонами ("Noise", "пустышка") через cosine similarity
- Збіги з порогом ≥ 0.80 автоматично відсіюються

**Крок 4.2: Інкрементне розпізнавання**
- Обличчя, що залишилися, порівнюються з іменованими кластерами (поріг 0.75)
- Збіги автоматично отримують ім'я існуючого кластера

**Крок 4.3: HDBSCAN-кластеризація**
- Тільки нові обличчя (без `cluster_id`) кластеризуються алгоритмом HDBSCAN
- Параметри: `min_cluster_size=3`, `metric='euclidean'`
- Кожному кластеру присвоюється унікальний ID `Cluster_N`

**Крок 4.4: Внутрішньокластерна дедуплікація**
- Всередині кожного кластера видаляються дубльовані обличчя (similarity ≥ 0.95)
- Запобігає появі одного й того самого обличчя 3-5 разів в одному кластері

**Крок 4.5: Графічний веб-інтерфейс**
- Eel-інтерфейс у браузері показує прев'ю всіх облич у нових кластерах
- Користувач вводить ім'я або натискає "Noise"/"пустышка"
- Імена зберігаються в полі `cluster_id` таблиці `faces`

**Результат:** Усі обличчя розподілені за іменованими кластерами або відфільтровані як шум.

#### Етап 5: Організація файлів

**Модуль:** `file_organizer.py` → `organize_files()`

1. Читання всіх записів з таблиці `media` з їхніми статусами та пов'язаними обличчями
2. **Логіка розподілу:**
   - 0 облич → `Пейзажі та Архітектура/Рік/Локація/`
   - Тільки "пустышка"/"Noise" → `Випадкові перехожі/Рік/Локація/`
   - ≥3 обличчя → `Групові фото/Рік/Локація/`
   - 1-2 розпізнаних обличчя → `Люди/Ім'я/Рік/` (копіюється в папку кожної людини)
   - 1-2 нерозпізнаних обличчя (`Cluster_X`) → `Невідомі особи/Рік/Локація/`
   - Скріншоти → `Кошик/Скріншоти та Меми/`
   - Дублікати → `Кошик/Візуальні дублікати/` або `Кошик/Дублікати MD5/`
3. **Фізичне копіювання** файлів із перевіркою MD5-хешу після копіювання
4. Створення ієрархії папок `Рік/Локація/` або `Рік/` залежно від наявності GPS-даних

**Результат:** Структурований архів у папці `output/` із файлами, розкладеними за категоріями.

### База даних SQLite

**Таблиця `media`:**
- `id` — унікальний ідентифікатор
- `filepath` — повний шлях до вихідного файлу
- `filename` — ім'я файлу
- `filesize` — розмір у байтах
- `media_type` — "image" або "video"
- `creation_date` — Unix timestamp з EXIF
- `md5_hash` — MD5-хеш вмісту
- `image_hash` — perceptual hash (pHash)
- `status` — "keep", "duplicate", "screenshot"
- `latitude`, `longitude` — GPS-координати
- `country`, `city` — геокодовані назви

**Таблиця `faces`:**
- `id` — унікальний ідентифікатор
- `media_id` — зовнішній ключ на `media.id`
- `bbox` — JSON з координатами bounding box `{"x1": ..., "y1": ..., "x2": ..., "y2": ...}`
- `face_embedding` — BLOB з 512 float32 (2048 байт)
- `cluster_id` — ім'я кластера ("Іван", "Марія") або "Cluster_N", "Noise", "пустышка"

### Алгоритми машинного навчання

**YOLOv8-Face:**
- Архітектура: YOLOv8n (nano) з модифікацією для детекції облич
- Вхідні дані: RGB-зображення довільного розміру
- Вихідні дані: bounding boxes + confidence scores
- Швидкість: ~100-200 FPS на GPU, ~5-10 FPS на CPU

**FaceNet (InceptionResnetV1):**
- Архітектура: Inception-ResNet v1 з triplet loss
- Вхідні дані: 160×160 RGB-зображення обличчя
- Вихідні дані: 512-вимірний L2-нормалізований вектор
- Попереднє навчання: VGGFace2 (3.3M зображень, 9131 людина)

**HDBSCAN:**
- Ієрархічна кластеризація на основі щільності
- Автоматичне визначення кількості кластерів
- Стійкість до шуму (outliers позначаються як `-1`)
- Метрика: Euclidean distance для 512-вимірних векторів

## Конфігурація

### Файл config.json

```json
{
  "source_dir": "input",
  "dest_dir": "output",
  "db_path": "data/database.db",
  "yolo_weights": "data/models/yolov8n-face.pt",
  "facenet_weights": "vggface2",
  "device": "cuda",
  "face_confidence": 0.6,
  "metadata_threads": 8,
  "screenshot_rules": {
    "keywords": ["screenshot", "screen", "скриншот", "снимка"],
    "check_exif": true,
    "aspect_ratio_min": 1.7,
    "aspect_ratio_max": 2.2
  },
  "duplicates": {
    "hash_size": 8,
    "hash_threshold": 4
  }
}
```

### Параметри командного рядка

| Параметр | Опис | Приклад |
|----------|----------|--------|
| `--config` | Шлях до файлу конфігурації | `--config my_config.json` |
| `--source` | Папка з вихідними медіа | `--source /mnt/photos` |
| `--dest` | Папка для організованого архіву | `--dest /mnt/organized` |
| `--db` | Шлях до бази даних SQLite | `--db /tmp/media.db` |
| `--device` | Пристрій для нейромереж | `--device cpu` або `--device cuda` |
| `--yolo-weights` | Шлях до ваг YOLOv8-Face | `--yolo-weights models/yolo.pt` |
| `--face-conf` | Поріг впевненості детекції | `--face-conf 0.7` |
| `--save-settings` | Зберегти параметри в config.json | `--save-settings` |
| `--reset-db` | Видалити БД та почати заново | `--reset-db` |

**Пріоритет налаштувань:** CLI-аргументи > config.json > дефолтні значення

### Налаштування дедуплікації

**Параметр `hash_threshold`** (за замовчуванням 4):
- **0-2** — дуже сувора дедуплікація (тільки майже ідентичні зображення)
- **3-5** — помірна (рекомендується, відсіває кадри із серійної зйомки)
- **6-10** — агресивна (може видалити схожі, але різні знімки)

### Налаштування детекції скріншотів

**Параметр `aspect_ratio_min` / `aspect_ratio_max`:**
- Для вертикальних екранів смартфонів: 1.7-2.2 (за замовчуванням)
- Для горизонтальних екранів ноутбуків: 1.3-1.8
- Вимкнути перевірку співвідношення сторін: встановіть обидва значення в 0

## Часті запитання

**Q: Програма не бачить мою GPU, хоча CUDA встановлена.**  
A: Переконайтеся, що версія PyTorch сумісна з вашою версією CUDA. Перевірте: `python -c "import torch; print(torch.cuda.is_available())"`. Якщо `False`, перевстановіть PyTorch із правильною версією CUDA із сайту pytorch.org.

**Q: Детекція облич пропускає деякі обличчя.**  
A: Знизьте поріг впевненості: `--face-conf 0.4`. Врахуйте, що це збільшить кількість помилкових спрацьовувань (статуї, постери).

**Q: Програма створює занадто багато кластерів для однієї людини.**  
A: Це може відбуватися через різне освітлення або вік на фотографіях. Після першого запуску назвіть усі кластери однієї людини однаковим ім'ям — при наступному запуску система об'єднає їх автоматично.

**Q: Чи можна запустити програму без графічного інтерфейсу?**  
A: Так, для повністю автоматичного режиму ви можете запустити конвеєр безпосередньо, імпортувавши функції з модулів `src/`. При цьому кластери залишаться неіменованими (`Cluster_N`).

**Q: Скільки місця займає база даних?**  
A: Приблизно 2-3 КБ на кожне фото (метадані + embeddings всіх облич). Для архіву з 10 000 фото — близько 20-30 МБ.

**Q: Програма зависла на етапі кластеризації.**  
A: HDBSCAN може працювати довго на великих датасетах (>10 000 облич). Перевірте логи в `data/logs/organizer.log`. Якщо процес дійсно завис, перезапустіть із `--reset-db`.
